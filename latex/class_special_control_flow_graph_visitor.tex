{\footnotesize\ttfamily void Special\+Control\+Flow\+Graph\+Visitor\+::build\+Dataflow\+Graph (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})}

This method uses the information gained in the pass of this visitor (i.\+e., calling visit(..) on a node), in particular the information of variable reads and writes, and constructs a data flow graph. This graph has an edge from each statement where a variable is read to the last statement where the respective variable was written. By considering \mbox{\hyperlink{class_if}{If}} statements, there might potentially exist multiple of such last writes to a variable. The following example demonstrates a program where variable access to \char`\"{}a\char`\"{} in line 06 would have two backward edges (to line 02 and line 04) as both statements could cause a write to variable \char`\"{}a\char`\"{}.

01\+: if (c $>$ 100) \{ 02\+: a = 1; 03\+: else \{ 04\+: a = 0; 05\+: \} 06\+: c = a$\ast$22;

The data flow graph is constructed in two passes\+: In the first pass, we traverse the control flow graph in a breadth-\/first search (BFS) style and propagate the knowledge at which node a all variables seen so far have been written the last time. This also considers joint points (i.\+e., nodes with two incoming edges) properly. In case that a loop is involved, it might be necessary to visit the loops body twice. In the second pass, we iterate over all nodes that we have visited in the first pass and check whether the respective node reads a variable. \mbox{\hyperlink{class_if}{If}} yes, we lookup when the respective variable was written the last time (i.\+e., in which node) and add an backward edge (last\+\_\+written\+\_\+node -\/$>$ read\+\_\+node). \mbox{\Hypertarget{class_special_control_flow_graph_visitor_ab35b2c221b2701c3ab2dd2f1ec41cf08}\label{class_special_control_flow_graph_visitor_ab35b2c221b2701c3ab2dd2f1ec41cf08}}
